# AutomationExercise – Manual Web Testing Project

This repository contains end-to-end **manual testing** performed on a real e-commerce web application (AutomationExercise).  
The project demonstrates professional QA practices including test scenarios, test cases, bug reporting, and requirement traceability.

---

## Application Under Test
**AutomationExercise.com** – An e-commerce demo website used for functional and UI testing.

---

##  Testing Scope
The following functionalities were tested:

- User Registration and Login
- Product Listing and Search
- Add to Cart and Cart Management
- Checkout Flow (simulation)
- Form Validations
- Navigation and UI Elements
- Error Handling and Negative Scenarios

---

##  Deliverables
This project includes the following QA documents:

- **Test Scenarios** – High-level testing coverage
- **Test Cases** – Detailed step-by-step functional test cases
- **Bug Report** – Identified defects with severity and priority
- **Requirement Traceability Matrix (RTM)** – Mapping requirements to test cases

All documents are organized in separate folders for clarity.

---

##  Testing Approach
- Functional Testing
- UI and Usability Testing
- Validation and Boundary Testing
- Negative Test Case Execution

---

## Tools Used
- Manual Testing Techniques
- MS Excel / Google Sheets
- GitHub for version control and documentation

---

##  Business Value
Manual testing helps identify functional and usability issues early, improving application quality, reducing production defects, and enhancing user experience.
